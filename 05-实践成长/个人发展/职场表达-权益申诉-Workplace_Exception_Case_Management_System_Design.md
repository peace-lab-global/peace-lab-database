# 职场异常场景处理系统模块设计 (Exception Case Management System Design)

> **定位**：为企业内部员工关系、HRBP、法务合规与业务管理者设计一套可落地的异常场景处置模块，统一处理孕产歧视投诉、报复性低绩效申诉、违法调岗争议、劝退协商纠纷等高敏感劳动关系事件
> **核心目标**：把零散、情绪化、被动应对的冲突处理，升级为可上报、可留痕、可调查、可协同、可预警、可复盘的全周期管理系统
> **适用边界**：以下内容偏产品方案与流程设计，不构成法律意见；具体规则需结合企业制度、数据合规要求与当地劳动争议处理实践落地

## 一、模块建设目标

本模块建议命名为：**异常场景处理中心**。

它不是一个单点“投诉箱”，而是一个覆盖以下全链路的业务系统：
1. **事件上报**：员工、HR、经理、工会或合规渠道发起事件登记
2. **证据收集**：结构化整理聊天、邮件、绩效、调岗、会议、录音、证人等材料
3. **风险分级**：自动判断是否涉及歧视、报复、程序瑕疵、违法变更劳动条件等高风险问题
4. **调查核实**：分配调查人、访谈相关方、形成事实时间线与核查意见
5. **解决方案生成**：输出调解、恢复、撤销、纠偏、补偿、保护性安排或升级处理建议
6. **执行跟踪**：跟进整改动作、回访员工、监控是否出现二次报复
7. **结案复盘**：沉淀案例、规则、经验，形成后续预警和建议能力

## 二、重点覆盖的异常场景

### 2.1 孕产歧视投诉
典型问题包括：
- 因怀孕、产假、哺乳、婚育计划被降薪、边缘化、卡晋升或撤项目
- 面试、录用、转正、晋升过程中出现婚育相关差别对待
- 产假返岗后被变相降级、调岗、削减职责

### 2.2 报复性低绩效申诉
典型问题包括：
- 员工投诉后被突然打低绩效、取消奖金、剥离项目
- 绩效标准临时变化，且缺少事前沟通与过程辅导记录
- 绩效结论与历史评价、实际交付、同岗位标准明显不一致

### 2.3 违法调岗争议
典型问题包括：
- 未经协商一致，单方变更岗位、地点、汇报线、薪酬结构
- 借组织调整名义进行事实降职、降薪或边缘化
- 调岗安排与劳动合同约定、岗位说明、实际业务需要明显冲突

### 2.4 劝退协商纠纷
典型问题包括：
- 以“主动离职更体面”为由持续施压
- 在未完成事实认定和程序说明前，直接推进离职协议签署
- 以PIP、调岗、架空、负面评价等方式迫使员工离开

## 三、设计原则

- **事实优先**：所有处理都围绕时间线、证据链、制度依据展开，而非围绕情绪站队
- **场景化分流**：不同争议类型使用不同核查清单、调查模板和解决方案库
- **最小必要知情**：谁该看什么、看到什么程度，必须严格按角色与权限控制
- **留痕可审计**：关键动作、结论、审批、导出、删除都应有日志
- **自动预警前置**：在争议升级前识别风险信号，减少事后补救成本
- **建议不替代判断**：系统可推荐处理路径，但最终结论由授权角色作出
- **兼顾员工体验**：界面简洁、表单短路径、状态透明，避免二次伤害

## 四、角色与权限设计

| 角色 | 典型用户 | 核心权限 |
| --- | --- | --- |
| 员工/申诉人 | 当事员工 | 提交事件、补充材料、查看本人案件状态、确认处理意见、发起异议 |
| 直属经理 | 业务管理者 | 在授权范围内提交说明、配合访谈、执行整改，不默认查看全部敏感材料 |
| HRBP | 人力BP | 受理案件、补证、协调沟通、查看本业务范围案件 |
| 员工关系/ER | 员工关系团队 | 主导调查、分级定性、组织会审、生成处理方案 |
| 法务/合规 | 法务、合规负责人 | 审核高风险案件、提供法规意见、控制外部风险 |
| 调查员 | 指定调查角色 | 查看调查所需材料、发起访谈、输出核查报告 |
| 工会/申诉委员会 | 员工代表机制 | 参与监督、出具意见、提供保护性支持 |
| 审批人 | HR负责人/业务负责人 | 审批处理结论、保护性措施、调岗修复、补偿方案 |
| 系统管理员 | 平台管理员 | 配置流程、角色、字段、预警规则，但不默认读取案件正文 |
| 审计角色 | 审计/监察 | 查看审计日志、流程合规性和脱敏报表 |

### 4.1 权限控制要点
- 采用 **RBAC + 数据范围控制 + 敏感字段脱敏** 的组合模式
- 孕产、医疗、心理、录音、身份证明等字段设为**高敏感等级**
- 业务经理默认只能看到与其整改责任有关的信息，不应直接访问完整投诉正文
- 所有下载、导出、打印操作必须记录操作人、时间、用途与审批单号

## 五、全周期流程设计

建议案件主状态如下：

`草稿` → `已提交` → `待初筛` → `待补证` → `已受理` → `调查中` → `风险会审中` → `方案待确认` → `执行中` → `回访观察中` → `已结案` → `复盘归档`

### 5.1 阶段一：事件上报

#### 入口设计
- 员工自助上报
- HR/工会代录入
- 匿名线索入口（仅适合线索预登记，不直接形成正式结论）
- 合规热线/邮件导入

#### 上报表单核心字段
- 事件类型：孕产歧视 / 报复性低绩效 / 违法调岗 / 劝退协商 / 其他
- 涉及人员：本人、被投诉对象、在场人员、证人
- 事件起止时间
- 主要发生地点/沟通渠道
- 核心事实摘要
- 当前影响：工资、岗位、绩效、项目、身心状态、出勤
- 是否存在紧急风险：报复、逼签、停权、停岗、停薪、人身威胁
- 已掌握证据清单
- 希望组织采取的措施

#### 系统动作
- 自动生成案件编号
- 自动判断是否进入高风险绿色通道
- 自动触发材料补全清单
- 自动推送保密提示与证据保存指南

### 5.2 阶段二：证据收集

系统应支持以下证据对象：
- 聊天记录
- 邮件
- 会议纪要
- 绩效文件
- 调岗通知/岗位说明
- 离职协商记录
- 录音录像
- 证人说明
- 合同、工资、考勤、社保等基础劳动关系材料

#### 证据管理能力
- 文件上传与批量归档
- 证据标签化：时间、来源、证明目的、真实性等级
- 证据链可视化：按时间线展示关键材料
- 重复材料识别
- 原始文件与摘要分层查看
- 证据缺口提示：如缺少书面调岗通知、缺少绩效辅导记录、缺少申诉后不利处理时间点对照等

### 5.3 阶段三：调查核实

#### 调查任务包
系统应自动生成调查任务包，至少包括：
1. 争议问题定义
2. 待核实事实清单
3. 应访谈对象名单
4. 待调取制度文件
5. 待补充证据缺口
6. 关键时间节点
7. 调查截止时间与SLA

#### 调查动作
- 发起访谈排期
- 形成访谈纪要
- 提交管理者说明
- 调取制度与历史绩效依据
- 对比同岗样本
- 判断是否存在程序缺失、差别对待、报复性关联

#### 调查输出
- 事实时间线
- 争议焦点
- 证据支持度评估
- 风险等级
- 初步责任归因
- 推荐处理路径

### 5.4 阶段四：解决方案生成

系统不直接代替HR或法务决策，但应输出**标准化方案包**：

#### 方案包结构
- 事实摘要
- 适用制度/法律依据
- 风险判断
- 可选方案A/B/C
- 对员工的保护性安排建议
- 对组织的整改要求
- 对业务管理者的执行动作
- 回访与观察周期

#### 典型方案类型
- 停止不当行为并书面确认边界
- 撤销不合理绩效结论或启动复核
- 暂停不当调岗并恢复原岗/原汇报线/原薪酬
- 引入第三方复核或升级会审
- 调整管理接口、设置隔离汇报或保护性安排
- 启动离职协商合规审查
- 启动纪律处分、培训整改或组织治理动作

### 5.5 阶段五：执行与回访

系统需支持：
- 整改任务拆分到人
- 到期提醒与逾期升级
- 回访问卷或回访纪要
- 二次报复观察期记录
- 处理结果确认与异议入口

### 5.6 阶段六：结案与案例沉淀

结案时应形成：
- 结案意见书
- 证据目录归档
- 处理方案执行结果
- 风险复盘结论
- 案例标签
- 是否纳入案例库

## 六、场景化处理流程

## 6.1 孕产歧视投诉流程

### 关键识别点
- 投诉是否发生在怀孕、产检、产假、哺乳或返岗节点前后
- 是否存在晋升、调薪、项目分配、录用转正的明显变化
- 是否出现婚育提问、岗位替代、边缘化言论等直接线索

### 重点证据
- 面试或沟通记录中的婚育相关问答
- 返岗前后职责变化记录
- 晋升、调薪、绩效口径变化材料
- 同岗位同层级比较样本

### 推荐方案
- 先做差别对待比对
- 再判断业务合理性与制度一致性
- 对高风险场景自动升级法务/ER联合审核

## 6.2 报复性低绩效申诉流程

### 关键识别点
- 申诉/举报后短期内出现低绩效、剥离项目、取消奖金
- 历史绩效长期正常，但争议节点后突然恶化
- 考核过程缺乏过程辅导、目标确认、面谈纪要或书面改善记录

### 重点证据
- 申诉时间与低绩效时间的前后关系
- 绩效目标版本变化记录
- 辅导记录、周报、项目产出
- 同类岗位评价标准对照

### 推荐方案
- 自动核查程序完整性
- 触发绩效复核路径
- 对疑似报复性案件限制单一管理者直接定案

## 6.3 违法调岗争议流程

### 关键识别点
- 是否未经协商一致即变更岗位核心内容
- 是否伴随薪酬、地点、职责、汇报关系的实质不利变化
- 是否有劳动合同、员工手册或组织架构调整依据支持

### 重点证据
- 劳动合同及岗位约定
- 原岗与新岗职责对比
- 调岗通知、会议纪要、沟通记录
- 工资、绩效、地点变化前后数据

### 推荐方案
- 先做“合理调配”与“实质变更”区分
- 对高风险调岗自动推送法务意见模板
- 输出恢复原岗、重新协商或补偿评估建议

## 6.4 劝退协商纠纷流程

### 关键识别点
- 是否存在高频次离职施压、逼签、孤立、停权、架空
- 是否在未完成调查与说明前直接要求签字
- 是否存在“不同意就低绩效/PIP/调岗”的联动施压

### 重点证据
- 协商邮件、微信、会议录音
- 离职协议版本与修改记录
- PIP、绩效、调岗、权限变更等联动材料
- 相关时间线与在场人员

### 推荐方案
- 强制进入高风险协商审查流程
- 对协议签署设置冷静期、法务审阅点和员工确认提示
- 对疑似逼迫离职案件自动生成风险提醒与升级路径

## 七、自动预警机制设计

系统应提供“规则预警 + 时间预警 + 组合预警”三层机制。

| 预警类型 | 触发信号 | 系统动作 |
| --- | --- | --- |
| 孕产歧视预警 | 员工处于孕期/产假/返岗阶段，且出现调岗、降薪、撤项目、晋升冻结 | 自动标记为高敏感案件，要求ER与法务共同审阅 |
| 报复性绩效预警 | 员工投诉后30/60/90天内出现低绩效、奖金取消、权限收回 | 自动拉取历史绩效对比并触发复核建议 |
| 违法调岗预警 | 调岗伴随工作地点、薪酬结构、职级或职责显著变化，且无协商记录 | 自动要求补充协商依据与合同条款 |
| 劝退施压预警 | 短时间内多次发起离职协商、签约催促、停权或项目剥离 | 自动升级为高风险劳动关系事件 |
| 超时预警 | 调查、审批、回访节点超出SLA | 自动提醒责任人并向上升级 |
| 二次报复预警 | 结案后观察期内再次出现不利绩效、边缘化、排班异常等 | 自动重启案件并提示复盘历史处理 |

### 7.1 风险评分建议
可采用规则分：
- 事件敏感度分
- 证据完整度分
- 报复概率分
- 程序缺失分
- 人员层级影响分
- 外部争议升级概率分

根据总分输出：`低风险 / 中风险 / 高风险 / 紧急风险`。

## 八、案例库与历史建议能力

案例库不是简单文档库，而应具备**检索、比对、推荐、复用**能力。

### 8.1 案例库字段
- 案例类型
- 行业/部门/岗位类型
- 涉及角色层级
- 主要争议点
- 证据构成
- 调查结论
- 处理方案
- 执行结果
- 复发情况
- 法律/制度依据
- 风险标签

### 8.2 推荐逻辑
当新案件进入系统时，自动推荐：
1. 相似案例
2. 证据缺口提示
3. 推荐访谈对象
4. 推荐处理路径
5. 相关模板文档
6. 典型风险点

### 8.3 建议引擎形式
建议优先采用**规则引擎 + 相似案例检索**的组合方式：
- 规则引擎负责合规底线和强制流程
- 相似案例检索负责经验性建议
- 高风险案件保留人工覆核，不做自动结论

## 十、界面设计建议

整体界面风格应采用：**低噪音、强结构、重状态、轻装饰**。

### 10.1 首页工作台
建议布局为三栏：
- 左侧：快捷入口（新建事件、上传证据、发起回访）
- 中间：案件列表与状态卡片
- 右侧：高风险预警、即将超时、相似案例建议

### 10.2 案件详情页
建议分为四个固定区域：
1. **页头摘要区**：案件编号、类型、风险等级、当前状态、责任人
2. **时间线区**：事件发生、申诉、补证、访谈、会审、处理、回访
3. **主工作区**：标签页切换“事实 / 证据 / 调查 / 方案 / 回访 / 日志”
4. **侧边动作区**：补证、指派、升级、导出、申请审批

### 10.3 证据页
- 默认卡片式展示
- 支持按时间、来源、证明目的筛选
- 支持“一键生成证据目录”
- 高敏感材料默认模糊预览，需要二次确认

### 10.4 案例库页
- 顶部筛选：事件类型、结果类型、部门、标签、法规依据
- 中部列表：案例摘要、相似度、处理结果
- 右侧推荐：适用模板、常见误区、制度建议

### 10.5 移动端体验
移动端仅建议开放：
- 事件上报
- 补充材料
- 查看本人案件状态
- 接收通知

涉及完整调查、批量导出、会审审批等高敏感操作，建议保留在桌面端。

## 十一、数据安全与合规要求

### 11.1 数据分级
建议至少分为：
- 一般案件信息
- 敏感劳动关系信息
- 高敏感个人信息
- 高敏感证据材料

### 11.2 安全控制
- 传输加密与存储加密
- 敏感字段脱敏显示
- 下载文件自动水印
- 高敏感材料访问需二次验证
- 导出、打印、分享需审批
- 证据文件保留原始哈希值用于防篡改校验

### 11.3 审计要求
- 全量记录访问、编辑、审批、导出日志
- 关键结论修改必须保留版本历史
- 删除仅允许逻辑删除，且需审批
- 高风险案件结论必须保留会审记录

### 11.4 隐私与最小化原则
- 非处理必需字段不采集
- 孕产、医疗、心理咨询等信息限制采集目的与查看范围
- 匿名线索与实名投诉应分开存储与流转

## 十二、处理建议生成逻辑

系统建议能力应主要输出四类结果：

1. **流程建议**：是否应进入高风险通道、是否需要法务参与、是否应启动保护性安排
2. **证据建议**：当前缺少哪些关键材料，哪些证据决定争议走向
3. **沟通建议**：推荐使用哪类正式通知、申诉模板、协商话术
4. **方案建议**：基于历史案例提供可参考的纠偏、调解、复核或补偿方案

### 12.1 建议输出示例
例如，当系统识别为“申诉后30天内低绩效 + 缺少辅导记录 + 同岗样本异常”时，可输出：
- 建议启动绩效复核
- 建议限制原评价人单独定案
- 建议补充历史周报、目标确认与面谈记录
- 建议参考：

## 十三、关键指标设计

建议跟踪以下指标：
- 案件受理时长
- 调查完成时长
- 高风险案件升级率
- 证据补齐率
- 处理方案执行完成率
- 结案后30/60/90天复发率
- 二次报复发生率
- 员工回访满意度
- 案件转外部仲裁/诉讼比例
- 组织制度修订触发次数

## 十四、分阶段落地建议

### 14.1 第一阶段：MVP
优先上线：
- 事件上报
- 证据上传
- 案件流转
- 调查任务
- 处理方案模板
- 基础权限与审计日志

### 14.2 第二阶段：增强版
新增：
- 自动预警
- 相似案例推荐
- 规则评分
- 回访观察期管理
- 组织风险看板

### 14.3 第三阶段：智能化版
新增：
- 案例标签自动建议
- 证据缺口自动识别
- 处理路径推荐
- 制度漏洞归因分析

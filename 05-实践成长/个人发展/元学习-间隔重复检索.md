> **学科定位**: 05-实践成长 > personal-development > meta-learning

## 一、间隔重复 (Spaced Repetition)

### 1.1 核心原理

| 概念 | 说明 |
|------|------|
| **遗忘曲线** | Ebbinghaus (1885): 记忆随时间指数衰减 |
| **间隔效应** | 分散学习 > 集中学习 (Cepeda et al., 2006 Meta) |
| **最优间隔** | 在即将遗忘时复习 → 记忆最大化 |
| **理论基础** | 存储强度 vs 提取强度 (Bjork & Bjork, 1992) |

### 1.2 间隔效应的科学证据

| 研究 | 发现 | 效应量 |
|------|------|--------|
| Cepeda et al. (2006) | 254项研究Meta: 间隔>集中, 跨所有材料类型 | d=0.42-1.12 |
| Karpicke & Bauernschmidt (2011) | 间隔检索比重读效果好3倍 | 大效应 |
| 医学教育Meta (2018) | 间隔重复在医学知识保持中效果显著 | d=0.75 |

### 1.3 间隔时间表

#### Leitner 盒子系统

```
盒子 1: 每天复习 (新卡/答错)
盒子 2: 每隔1天
盒子 3: 每隔3天
盒子 4: 每隔7天
盒子 5: 每隔14天
盒子 6: 每隔30天

答对 → 升一级
答错 → 回到盒子1
```

#### SM-2 算法 (SuperMemo)

| 参数 | 说明 |
|------|------|
| 初始间隔 | 1天 → 6天 |
| 后续间隔 | 上次间隔 × 难度因子 (EF) |
| 难度因子 | 2.5 (默认), 根据回答质量调整 |
| 最低EF | 1.3 (非常困难的卡片) |

### 1.4 FSRS 算法 (Anki新一代)

| 特性 | 说明 |
|------|------|
| 全称 | Free Spaced Repetition Scheduler |
| 核心 | 基于机器学习预测遗忘概率 |
| 优势 | 比SM-2更准确，适应个人记忆模式 |
| 参数 | 自动根据用户数据优化 |

## 二、检索练习 (Retrieval Practice)

### 2.1 测试效应 (Testing Effect)

| 核心发现 | 来源 |
|---------|------|
| 测试 (主动回忆) 比重读学习效果好50-150% | Roediger & Karpicke (2006) |
| 即使答错, 测试后再学习的效果也优于直接学习 | Kornell et al. (2009) |
| 效果跨年龄段 (儿童到老年) 和材料类型一致 | Rowland (2014) Meta |

### 2.2 检索练习的机制

| 机制 | 说明 |
|------|------|
| **编码精细化** | 检索过程中建立新的记忆路径 |
| **提取增强** | 每次成功检索增强提取路径 |
| **元认知校准** | 暴露真实记忆状态, 纠正虚假熟悉感 |
| **语义整合** | 检索促进新旧知识的连接 |

### 2.3 检索练习的实施形式

| 形式 | 方法 | 适用场景 |
|------|------|---------|
| **闪卡 (Flashcard)** | 正面问题/反面答案 | 事实性知识 |
| **自由回忆** | 合上书本写出所有记得的 | 概念框架 |
| **练习测试** | 做题/模考 | 应试 |
| **教学** | 向他人讲解 | 深度理解 |
| **脑图绘制** | 从记忆中画概念图 | 知识结构 |
| **康奈尔笔记** | 用左栏问题测试右栏内容 | 课堂学习 |

## 三、Anki 系统详解

### 3.1 Anki 核心原理

| 组件 | 功能 |
|------|------|
| 卡片 (Card) | 知识的最小单元, 一问一答 |
| 牌组 (Deck) | 卡片集合, 按主题组织 |
| 调度器 (Scheduler) | SM-2/FSRS 算法安排复习时间 |
| 回答等级 | Again/Hard/Good/Easy → 调整下次间隔 |

### 3.2 高质量卡片制作原则 (20 Rules by Piotr Wozniak)

| 原则 | 说明 | 示例 |
|------|------|------|
| **最小信息原则** | 每张卡片只含一个知识点 | 一个事实/一个概念 |
| **理解先于记忆** | 先理解再做卡片 | 不做不理解的卡片 |
| **图片优先** | 图像记忆优于文字 | 加入示意图/图表 |
| **使用挖空 (Cloze)** | 句子中挖空比Q&A更自然 | "线粒体是细胞的{{c1::能量工厂}}" |
| **个人化** | 加入个人经验/情感 | 个人例子 |
| **避免集合** | 不要一张卡片列多个项目 | 拆成多张 |
| **设置上下文** | 避免模糊的问题 | 加标签/来源 |

### 3.3 Anki 工作流

| 阶段 | 操作 | 时间 |
|------|------|------|
| **输入** | 学习新材料, 做笔记 | 不限 |
| **制卡** | 将笔记转化为Anki卡片 | 学习后当天 |
| **新卡学习** | 每天学习20-30张新卡 | 10-15分钟 |
| **复习** | 完成当日到期复习 | 随时间累积 |
| **维护** | 标记/编辑/删除问题卡片 | 持续 |

## 四、交替练习 (Interleaving)

### 4.1 交替 vs 集中练习

| 维度 | 集中练习 (Blocking) | 交替练习 (Interleaving) |
|------|-------------------|----------------------|
| 方式 | AAABBBCCC | ABCBCAACB |
| 短期表现 | 更好 (流畅感高) | 较差 (困难感高) |
| 长期保持 | 较差 | 更好 (d=0.43) |
| 迁移 | 较差 | 更好 |
| 适用 | 完全陌生的材料入门 | 需要区分类似概念 |

### 4.2 交替练习的应用

| 场景 | 集中练习 | 交替练习 |
|------|---------|---------|
| 数学 | 连做30题加法 | 加减乘除混合 |
| 音乐 | 一首曲子练到熟 | 3首交替练习 |
| 运动 | 只练正手100次 | 正手/反手/发球混合 |
| 语言 | 一次只学动词 | 动词/名词/形容词混合 |

## 五、精加工策略 (Elaboration)

### 5.1 精加工问询 (Elaborative Interrogation)

| 核心问题 | 作用 |
|---------|------|
| **为什么这是真的?** | 激活因果推理 |
| **这与我已知的有什么关系?** | 建立知识网络 |
| **有什么例外情况?** | 深化理解 |
| **这让我想到什么类比?** | 创建记忆钩子 |

### 5.2 双重编码 (Dual Coding)

| 编码 | 形式 | 组合效果 |
|------|------|---------|
| 语言编码 | 文字、言语 | 中 |
| 视觉编码 | 图像、图表 | 中-高 |
| **双重** | 文字+图像同时 | **最佳** (相加效应) |

### 5.3 具象化策略

| 方法 | 操作 | 效果 |
|------|------|------|
| **记忆宫殿** | 将信息放置在熟悉空间的位置 | 序列记忆极佳 |
| **关键词法** | 外语词→谐音→视觉意象 | 词汇学习有效 |
| **故事法** | 将无关项目编成故事 | 列表记忆有效 |
| **PQ4R** | Preview-Question-Read-Reflect-Recite-Review | 教材学习结构化 |

| 笔记法 | 检索整合方式 |
|--------|------------|
| **康奈尔笔记** | 左栏→自测问题; 右栏→答案 |
| **Zettelkasten** | 每条笔记→关联检索; 永久笔记→Anki卡片 |

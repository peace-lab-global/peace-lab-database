# 跨部门协作冲突处理 (Cross-Functional Conflict Resolution)

> **定位**：帮助处理跨部门协作中的责任不清、优先级冲突、信息不对称和甩锅问题
> **核心目标**：先对齐目标，再界定责任；先留痕，再升级；把情绪冲突转成流程与事实管理

## 一、常见冲突类型

- 谁负责不清，事情掉在地上
- 多部门都说“这不是我的事情”
- 优先级不同，彼此互相卡住
- 临近截止日才暴露依赖问题
- 出问题后互相甩锅

## 二、责任界定方法

### 2.1 四问法
每个跨部门事项至少问清四个问题：
1. **目标是什么**：最终要交付什么结果
2. **负责人是谁**：谁是唯一Owner
3. **依赖项是什么**：需要谁在什么时间前提供什么
4. **风险怎么报**：延期、变更、卡点由谁何时上报

### 2.2 简版责任表
```text
事项：________
最终Owner：________
我方负责：________
对方负责：________
交付时间：________
风险上报节点：________
```

### 2.3 防甩锅原则
- 没有Owner的任务，等于没人负责
- 没有时间点的承诺，等于未承诺
- 没有书面确认的分工，容易事后改口

## 三、沟通话术模板

### 3.1 对齐目标
- "为了避免各自理解不一致，我们先确认这件事的最终目标。"
- "我理解当前共同目标是________，请大家确认。"

### 3.2 界定责任
- "为了推进顺畅，我们把我方、你方和唯一Owner先定下来。"
- "这部分我方可以承担________，需要你方在________前提供________。"

### 3.3 处理推诿
- "如果这部分暂不由你负责，请明确当前接口人是谁。"
- "如果当前Owner还未确定，我建议今天先定唯一负责人。"

### 3.4 处理冲突升级
- "我们先不讨论谁更忙，先确认哪一个节点最影响整体交付。"
- "如果今天无法直接达成一致，我建议把分歧点整理后请上级拍板。"

## 四、邮件留证技巧

### 4.1 邮件必须写清四件事
- 当前结论
- 分工边界
- 时间点
- 风险与待确认事项

### 4.2 标准邮件模板
```text
主题：关于X事项的分工与时间确认

各位好，
为确保X事项按时推进，当前确认如下：
1. 目标：________；
2. 我方负责：________；
3. 你方负责：________；
4. 关键时间点：________；
5. 当前风险：________；
6. 待确认事项：________。

如有偏差，请于________前回复；如无异议，我们将按以上内容推进。
```

### 4.3 留证技巧
- 用标题体现主题和事项
- 用编号列清责任和时间点
- 关键分歧一定落邮件，不只停留在口头或群聊
- 对重大变更及时发“更新版确认”

## 五、向上汇报策略

### 5.1 什么情况下要向上汇报
- 对方多次不响应
- 责任边界长期不清
- 冲突已影响整体交付
- 分歧不是执行问题，而是优先级和资源问题

### 5.2 向上汇报结构
- **事实**：目前卡在哪里
- **影响**：不处理会影响什么
- **分歧点**：双方具体卡在什么
- **建议**：希望上级拍什么板

### 5.3 向上汇报模板
```text
当前X事项推进受阻，主要卡点在跨部门责任和时间安排。
目前事实是：________。
如果今天不明确，可能影响________。
当前分歧点有两项：________。
我建议请您确认唯一Owner及最终时间点，我会据此继续推进。
```

## 六、实操步骤

1. 先用四问法界定目标、Owner、依赖和风险节点
2. 口头沟通后立刻邮件确认
3. 遇到推诿时不争辩动机，只追问责任和时间
4. 多次无效后，用“事实+影响+建议”向上汇报
5. 对关键事项形成更新版纪要或邮件

## 七、常见错误

- 只讲态度问题，不讲责任边界
- 只在群里催，不发正式确认
- 事情已经延期才第一次向上汇报
- 把跨部门冲突个人化

## 核心内容与观点 (Core Content & Perspectives)

### GTD核心理念

| 概念 | 描述 | 为何重要 |
| :--- | :--- | :--- |
| **心如止水** | 大脑清空所有待办事项，专注于当前 | 减少焦虑，提高专注 |
| **开放式循环** | 未完成的任务会占用心理资源 | 必须"关闭"这些循环 |
| **外部化** | 将所有待办事项移出大脑 | 大脑不是好的存储工具 |
| **下一步行动** | 明确每个项目的下一个具体物理行动 | 消除模糊带来的拖延 |

### GTD五步流程

| 步骤 | 英文 | 描述 | 关键问题 |
| :--- | :--- | :--- | :--- |
| **1. 收集** | Capture | 将所有事项收集到收件箱 | "这可能需要做什么？" |
| **2. 明确** | Clarify | 明确每个事项的含义和行动 | "这是什么？需要行动吗？" |
| **3. 组织** | Organize | 将事项放入合适的清单 | "它属于哪个类别？" |
| **4. 回顾** | Reflect | 定期审视所有清单 | "我的承诺是什么？" |
| **5. 执行** | Engage | 选择并执行行动 | "现在做什么最合适？" |

### 收集流程详解

| 收集工具 | 示例 | 要点 |
| :--- | :--- | :--- |
| **物理收件箱** | 办公桌收件盘 | 所有纸质物品 |
| **数字收件箱** | 邮件、笔记应用 | 所有数字输入 |
| **笔记本** | 随身携带的记录工具 | 随时捕捉想法 |
| **录音** | 语音备忘录 | 开车或不便书写时 |

### 明确流程 (处理决策树)

```
收件箱项目
    ↓
这是什么？
    ↓
需要行动吗？
    ├→ 否 → 垃圾 / 参考资料 / 将来/也许
    └→ 是 → 下一步行动是什么？
              ├→ 2分钟内能完成？ → 立即做
              ├→ 应该委托给他人？ → 委派
              └→ 需要延后？ → 放入日历或下一步清单
```

### 组织系统 (清单类型)

| 清单 | 内容 | 回顾频率 |
| :--- | :--- | :--- |
| **日历** | 特定日期/时间的行动和信息 | 每天 |
| **下一步行动清单** | 按情境分类的下一步行动 | 持续 |
| **项目清单** | 所有需要多步骤完成的目标 | 每周 |
| **等待清单** | 委托给他人的事项 | 每周 |
| **将来/也许清单** | 目前不做但可能想做的事 | 每周 |
| **参考资料** | 可能需要的信息 | 按需 |

### 情境清单示例

| 情境 | 描述 | 示例行动 |
| :--- | :--- | :--- |
| **@电脑** | 需要电脑才能做的事 | 写报告、发邮件 |
| **@电话** | 需要打电话的事 | 预约、咨询 |
| **@办公室** | 在办公室才能做的事 | 与同事讨论 |
| **@外出** | 出门时可以做的事 | 买东西、取快递 |
| **@家** | 在家才能做的事 | 家务、维修 |
| **@无网络** | 不需要网络的事 | 阅读、写作 |

### 每周回顾

| 步骤 | 操作 | 目的 |
| :--- | :--- | :--- |
| **清空收件箱** | 处理所有输入 | 确保没有遗漏 |
| **回顾日历** | 查看过去和未来 | 识别待办和准备 |
| **回顾项目清单** | 检查每个项目 | 确保每个项目有下一步行动 |
| **回顾等待清单** | 检查委托事项 | 跟进和提醒 |
| **回顾将来/也许** | 评估是否激活 | 保持可能性开放 |
| **头脑清空** | 写下新想法 | 保持大脑空净 |

### 自然计划法

| 步骤 | 描述 | 示例 |
| :--- | :--- | :--- |
| **1. 定义目的和原则** | 为什么要做？ | 举办成功的会议 |
| **2. 展望成果** | 成功是什么样子？ | 所有人积极参与 |
| **3. 头脑风暴** | 所有可能的想法 | 场地、议程、人员... |
| **4. 组织整理** | 按逻辑排序 | 时间线、责任分配 |
| **5. 明确下一步** | 具体的下一个行动 | 预订会议室 |

### 选择行动的四标准

| 标准 | 问题 | 考量 |
| :--- | :--- | :--- |
| **1. 情境** | 我现在在哪里？有什么工具？ | 只能看适合当前情境的行动 |
| **2. 时间** | 我有多少时间？ | 选择能在可用时间内完成的 |
| **3. 精力** | 我现在的精力状态？ | 高精力做难事，低精力做简单事 |
| **4. 优先级** | 什么最重要？ | 考虑价值和紧迫性 |

## 实践建议 (Practical Recommendations)

### GTD启动清单

| 步骤 | 操作 | 工具建议 |
| :--- | :--- | :--- |
| **1** | 准备物理和数字收件箱 | 文件盘、笔记应用 |
| **2** | 收集所有"开放式循环" | 大脑清空、抽屉整理 |
| **3** | 处理每一项直到收件箱清空 | 2小时专注时间 |
| **4** | 建立清单系统 | 任务管理应用 |
| **5** | 设置每周回顾时间 | 日历固定时间 |

### 推荐工具

| 工具类型 | 建议 | 特点 |
| :--- | :--- | :--- |
| **任务管理** | Todoist、OmniFocus、Things | 支持项目、情境、日期 |
| **笔记收集** | Notion、Evernote | 快速捕捉想法 |
| **日历** | Google Calendar | 时间块管理 |
| **参考资料** | Notion、文件夹系统 | 易于检索 |

### 与其他实践的整合

| 整合领域 | 结合方式 | 增强效果 |
| :--- | :--- | :--- |
| **深度工作** | GTD清理浅层工作，保护深度时间 | 减少干扰 |
| **习惯养成** | 将GTD流程习惯化 | 自动化任务管理 |
| **七个习惯** | GTD作为习惯3（要事第一）的工具 | 执行层面支持 |

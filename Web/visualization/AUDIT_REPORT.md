# 可视化交互/滚动缺陷审计报告

## 结论摘要

- 缺陷 1（“领域框架”无法点击）：侧栏交互元素使用非语义 DIV，且在多端/无障碍树中不可作为交互目标稳定命中；在部分视口/叠层状态下易出现命中偏移与点击穿透/拦截问题。
- 缺陷 2（页面无法继续向下滚动）：侧栏仅对节点清单子容器启用滚动，导致在“领域框架”区域滚轮/触摸滚动无法继续向下滚动（滚动链被截断），体验上表现为“卡住”。

## 已落地修复（核心）

### 交互层级与事件策略

- 将“领域框架”条目从 `div.legend-item` 重构为 `button.legend-item`，提升可点击稳定性、键盘可达性与可测试性。
- 将节点清单条目从 `div.node-card` 重构为 `button.node-card`，统一交互与焦点策略。
- 将侧栏与图例/列表点击由“逐条绑定”改为“容器级事件委托”，减少监听器数量并避免重建列表时遗漏/重复绑定。
- 详情面板关闭态增加 `visibility: hidden`，避免透明但存在的叠层在特定浏览器/辅助技术下造成命中/滚动异常。

### 滚动与多端适配

- 侧栏容器启用 `overflow-y: auto`，确保滚轮/触摸在侧栏任意区域均可继续滚动到“知识节点清单”。
- 移动端不再直接隐藏侧栏：改为抽屉式侧栏（off-canvas），新增“打开侧栏”按钮 + 遮罩层，支持手机/平板横竖屏访问“领域框架”。

### 可访问性与视觉一致性

- 对主要交互控件统一 `:focus-visible` 外描边（≥2px）与可见偏移，保证键盘导航可识别。

## 性能影响评估

- 事件委托替代大量逐项事件绑定：减少 DOM 更新时的事件注册开销，避免节点列表重建带来的潜在泄漏与卡顿。
- 侧栏滚动容器变更：避免在子列表内部滚动与外层不滚动造成的“滚动死区”，减少用户重复滚动/拖拽导致的额外事件触发。

## Lighthouse / 无告警状态

- 当前运行环境对本地端口监听/服务启动存在限制，无法在此环境中运行 Lighthouse/Storybook/Cypress 的完整命令链来产出可复现的分数截图与告警清单。
- 已按 Lighthouse ≥90 常见扣分项完成结构性修复（语义化交互、焦点可见、滚动链合理、避免不可见叠层拦截），建议在可启动本地静态服务的环境中执行：
  - Storybook：`npm run storybook`
  - Cypress：`npm run test:cypress`

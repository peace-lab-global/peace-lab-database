# 冥想台本制作标准流程

> **适用范围**：Peace Lab 所有录制级冥想引导词的制作、评估与交付
> **经验来源**：胜利冥想（Vijaya Victory Meditation）v2.0→v2.2 四轮迭代实证
> **文档层级**：流程规范（非单一脚本文档）

---

## 一、需求分析与转化流程

### 1.1 需求收集标准模板

每份新台本启动前，须完成以下需求卡：

```yaml
# ── 冥想台本需求卡 ──
project_name: ""          # 项目内部代号
series: ""                # 所属系列（如"内在力量系列"）
series_number:            # 系列编号

# ── 核心定义 ──
theme: ""                 # 核心主题（如"胜利""慈悲""放下"）
duration_target: ""       # 目标时长（如 15min）
target_audience: ""       # 目标人群（如"需要建立信心者、自我怀疑时刻"）
core_objective: ""        # 一句话核心目标
design_philosophy: ""     # 设计理念（用比喻描述风格基调）

# ── 声音设计 ──
voice_type: ""            # 声音类型（如"醇厚男声、胸腔共鸣"）
speech_rate: ""           # 语速范围（字/分钟）
music_style: ""           # 背景音乐风格（调性、节奏、乐器）
music_bpm: ""             # 音乐BPM范围

# ── 结构要求 ──
structure_type: ""        # 结构类型（三阶段/五阶段/自由式）
stages: []                # 各阶段名称与预估时长
anchor_points: []         # 核心身体锚点（如"脊柱觉知+心跳感知"）

# ── 约束条件 ──
posture_requirement: ""   # 体位要求（椅坐+盘腿/站姿/躺姿）
contraindications: []     # 禁忌症
delivery_formats: []      # 交付方式（录音/一对一/团体/直播）

# ── 质量要求 ──
target_version: ""        # 交付版本目标
pronunciation_support: "" # 是否需要发音标注（是/否）
```

### 1.2 需求转化四步法

```
需求收集 → 可行性评估 → 结构设计 → 制作排期
```

| 步骤 | 输入 | 输出 | 关键判断 |
|------|------|------|----------|
| 1. 收集 | 客户口头/文字需求 | 填写需求卡 | 区分"想要"与"需要" |
| 2. 评估 | 需求卡 | 可行性报告 | 时长是否合理、结构是否支撑主题 |
| 3. 设计 | 可行性报告 | 结构大纲 | 确定阶段划分与锚点设计 |
| 4. 排期 | 结构大纲 | 里程碑计划 | 初稿→评审→修订→终审→录制 |

### 1.3 需求优先级排序（P0-P3）

| 优先级 | 含义 | 示例 |
|--------|------|------|
| **P0** | 安全性——不做会造成伤害 | 安全退出通道、禁忌症标注 |
| **P1** | 核心体验——不做会影响冥想效果 | 三阶段结构完整性、身体锚点设计 |
| **P2** | 录制适配——不做会影响录音质量 | 前后鼻音处理、口语化、停顿设计 |
| **P3** | 增强体验——做了更好 | 变体建议、日常锚点植入、系列关联 |

### 1.4 可行性评估检查清单

- [ ] 主题是否有明确的心理学/冥想理论支撑？
- [ ] 目标时长与结构复杂度是否匹配？
- [ ] 目标人群是否有安全风险（如创伤、躁狂）？
- [ ] 声音设计与主题是否一致（如力量型不适合空灵女声）？
- [ ] 音乐BPM与语速是否可协调？
- [ ] 是否有现成参考资料或需从零研发？

## 二、台本评估方式

### 2.1 七维评估体系

每份台本须经过以下七个维度的系统评估，采用10分制打分：

| # | 维度 | 权重 | 通过线 | 评估焦点 |
|---|------|------|--------|----------|
| 1 | 三阶段结构与专业性 | 20% | ≥8 | 阶段完整性、时长分配、设计意图清晰度 |
| 2 | 语言流畅性/声音适配 | 20% | ≥8 | 呼吸组长度、绕口检测、节奏感 |
| 3 | 体位兼容性 | 15% | ≥9 | 所有身体指令是否中性化 |
| 4 | 口语化程度 | 15% | ≥8 | 书面语残留、定义式表达、连词堆叠 |
| 5 | 前后鼻音标注 | 10% | ≥9 | 对照表完整性、列位准确性 |
| 6 | 技术标注完整性 | 10% | ≥9 | 语气/情绪/音乐/停顿/音量覆盖率 |
| 7 | 版本记录 | 10% | ≥9 | YAML同步、变更说明完整 |

**总分通过线**：≥8.5/10 方可进入录制

### 2.2 维度一：三阶段结构与专业性

**检查要素**：

- 每阶段是否有明确的设计意图（不是简单堆砌内容）
- 阶段间是否有自然过渡（非硬切）
- 时长分配是否合理（开场≤2min，主体各≥3min，收束≥1.5min）
- 是否包含安全退出通道（P0）
- 高能量段后是否有落地过渡

**胜利冥想范式**（三乐章结构）：

| 阶段 | 时长 | 设计意图 |
|------|------|----------|
| 开场·落地 | ~1.5min | 快速锚定身体、建立安全感 |
| 第一乐章·回顾 | ~4min | 唤起事实性记忆（非空想） |
| 第二乐章·直面 | ~4min | 清醒面对（内嵌安全退出） |
| 第三乐章·宣告 | ~4min | 深沉确认（非口号式） |
| 收束·带走 | ~2min | 存入身体+日常锚点 |

### 2.3 维度二：语言流畅性/声音适配

**评估标准**（以醇厚男声、80-95字/分钟为基准）：

| 检查项 | 标准 | 典型问题 |
|--------|------|----------|
| 呼吸组长度 | ≤10字/气口 | 超长定语从句 |
| 节奏感 | 有变化（长短交替） | 匀速平铺 |
| 绕口检测 | 大声朗读无卡顿 | 多个zh/ch/sh连续 |
| 语气标注唯一性 | 每句比喻不重复 | "温暖""温柔"反复出现 |
| 句尾节奏 | 稳稳落地或微微上扬 | 拖沓/下坠 |

**语言流畅性红线**：
- 双虚词连续出现（"似乎仿佛"）
- 定语超过三层嵌套
- 书面连词（因此/然而/不仅...而且）
- 教科书式定义（"所谓XX，就是..."）

### 2.4 维度三：体位兼容性

**核心原则**：所有身体锚点描述须同时支持椅子坐姿与盘腿坐姿。

| 禁用表达 | 替代表达 | 原因 |
|----------|----------|------|
| "双脚踩地" | "身体的重量稳稳落在下方" | 盘腿无法踩地 |
| "感受脚底与地面的接触" | "感受身体坐在这里的重量" | 同上 |
| "回到双脚" | "回到身体坐稳的感觉" | 教学要点一致性 |
| "站稳脚跟" | "稳稳坐在这里" | 坐姿语境 |

**检查流程**：
1. 全文搜索"脚""踩""站""踏"等字眼
2. 逐一判断：物理指令 vs 隐喻用法
3. 物理指令全部中性化；隐喻（如"前行"）保留但加注释

### 2.5 维度四：口语化程度

**口语化核心准则**：
- 写给嘴巴，不是写给眼睛
- 每句话用说的，不是用读的
- 听众不会回看，必须一遍就懂

**检查清单**：

| # | 检查项 | 合格标准 |
|---|--------|----------|
| 1 | 嵌入式引号 | 全部转为间接引语或独立设置 |
| 2 | 书面连词 | 零出现 |
| 3 | 被动语态 | ≤1处/千字 |
| 4 | 四字成语 | 仅在韵律需要时使用 |
| 5 | 定义式表达 | 禁止（如"本身就是力量的表现"→"本身就是力量"） |
| 6 | 双虚词重复 | 禁止 |
| 7 | 语义精准性 | 每个形容词必须不可替换（如"更真实"→"更笃定"） |

**嵌入式引号转化规则**：

```
❌ 原文：那个总是说"你不行"的声音
✓ 改为：那个总说你不行的声音

❌ 原文：告诉自己"我可以"
✓ 改为：在心里确认——我可以（用破折号+换行替代引号）
```

### 2.6 维度五：前后鼻音标注

**标记体系**：
- ◇ = 前鼻音（-n结尾：an/en/in/un/ün）
- ◆ = 后鼻音（-ng结尾：ang/eng/ing/ong）

**检查要素**：
- 最危险对比组是否标注（≥5组）
- 逐段关键词表是否完整覆盖所有段落
- 表格列位是否准确（前鼻音在中列，后鼻音在右列）
- 录制口诀是否包含发音方法描述

**常见列位错误**（实践中发现的高频问题）：
- "冥(míng)" 是后鼻音◆，不是前鼻音
- "挺(tǐng)" 是后鼻音◆，不是前鼻音
- "脏(zàng)" 是后鼻音◆，不是前鼻音

### 2.7 维度六：技术标注完整性

**标注类型与覆盖率要求**：

| 标注类型 | 格式规范 | 覆盖率要求 |
|----------|----------|------------|
| 语气方向 | `[语气：比喻式描述]` | 每句100% |
| 情绪基调 | `[情绪：画面式描述]` | 每段100% |
| 音乐配合 | `[音乐提示：调性/音区/节奏]` | 每段100% |
| 停顿时长 | `[停顿 Xs——钢琴说明]` | 每处100% |
| 音量动态 | ①正常 ②轻声 ③加力 | 关键句100% |
| 呼吸提示 | `[此处自然吸气，麦克风捕捉]` | 关键点≥3处 |
| 安全机制 | 内嵌退出通道 | ≥1处 |

**标注语言统一规则**：
- 全部使用中文方括号 `[...]`
- 语气标注用比喻锚点，不用抽象形容词
- 停顿必须注明钢琴的配合方式
- 音量标注使用①②③三档数字编号

**语气标注质量要求**：

```
❌ 差：[语气：温暖]  ← 抽象、不可执行
✓ 好：[语气：如晴天下稳步行走的男人]  ← 画面感、可模仿

❌ 差：[语气：坚定]  ← 每句都能用
✓ 好：[语气：如最后一颗钉子敲入木板]  ← 唯一、精准
```

### 2.8 维度七：版本记录

**必须同步更新的位置**：
1. YAML前言的 `last_updated` 字段
2. 文件底部"版本记录"表格
3. 文件末尾版本标记行
4. INDEX.md 中的版本标注

## 三、制作工艺标准

### 3.1 台本结构规范

**标准章节顺序**：

```
1. YAML 前言
2. 标题 + 元信息摘要块
3. 理论基础
4. 实施指南（环境/语调/适用/禁忌/交付）
5. 完整引导词
   ├── 开场·落地
   ├── 第一乐章 [主题名]
   ├── 第二乐章 [主题名]
   ├── 第三乐章 [主题名]
   └── 收束·带走
6. 执行师教学要点
   ├── 常见反应与回应策略
   ├── 变体建议
   └── 搭配建议
7. 录制规格
8. 录制发音：前后鼻音速查
9. 版本记录
10. 相关链接
```

### 3.2 录制规格标准

| 参数 | 标准值 | 说明 |
|------|--------|------|
| 总时长 | 14:30-15:30 (15min版) | 根据需求卡调整 |
| 语速 | 80-95 字/分钟 | 配合音乐节奏 |
| 采样率 | ≥ 48kHz / 24bit | 最低标准 |
| 背景音乐 | 按需求卡设计 | 不抢人声 |
| 人声与音乐比例 | 人声占主导 | 音乐为人声的 30-40% |
| 停顿处理 | 音乐自然填充 | 不完全静默 |
| 后期处理 | 人声微量低频增强 +1.5dB@100-250Hz | 增加胸腔共鸣感 |
| 呼吸声 | 保留自然呼吸声 | 增加真实感和亲密感 |
| Room Reverb | 轻微 | 营造空间感但不模糊 |

### 3.3 前后鼻音标注制作方法

**制作步骤**：

1. **提取全文关键发音词**：逐句标记含 -n/-ng 韵尾的字词
2. **归类到对照表**：前鼻音◇放中列，后鼻音◆放右列
3. **识别最危险对比组**：选出同一段落内前后鼻音易混淆的对（≥5组）
4. **逐段整理**：按脚本结构分段，每段一个表格
5. **编写录制口诀**：用发音部位描述辅助记忆
6. **交叉验证**：逐字核查拼音归类是否正确（重点检查 ing/in、eng/en、ang/an）

**表格格式规范**：

```markdown
| 句子关键词 | 前鼻音◇ | 后鼻音◆ |
|---|---|---|
| [句子摘要] | 字(pīnyīn) 字(pīnyīn) | 字(pīnyīn) 字(pīnyīn) |
```

**高频错误防范**：
- `-ing` 结尾（如 míng, tǐng, dìng）是**后鼻音**，不是前鼻音
- `-iang` 结尾（如 liàng, qiáng）是**后鼻音**
- `-uan` 结尾（如 nuǎn, guān）是**前鼻音**
- `-ong` 结尾（如 zhōng, dòng）是**后鼻音**

### 3.4 版本控制规范

**版本号规则**：

| 版本类型 | 格式 | 触发条件 |
|----------|------|----------|
| 大版本 | vX.0 | 风格重构/结构重组 |
| 小版本 | vX.Y | 系列化升级/精细化审校 |

**版本记录表格模板**：

```markdown
| 版本 | 日期 | 说明 |
|---|---|---|
| v1.0 | YYYY-MM | 首版完成，含[包含内容摘要] |
| v2.0 | YYYY-MM | [重大变更描述] |
```

**每次版本更新必做**：
1. 更新 YAML `last_updated`
2. 在版本记录表添加新行（说明字段≤80字）
3. 更新文件末尾版本标记
4. 更新 INDEX.md 对应条目

## 四、质量保证流程

### 4.1 专业评审修复检查点

**审校流程**（四轮递进）：

```
第一轮：结构与安全审查（P0+P1）
    └── 安全退出通道 → 阶段完整性 → 时长分配

第二轮：语言与声音审查（P2）
    └── 口语化 → 绕口检测 → 体位兼容 → 鼻音标注

第三轮：技术标注审查（P2）
    └── 语气/情绪/音乐/停顿覆盖率 → 音量动态 → 呼吸提示

第四轮：终审与交叉验证
    └── 七维打分 → 教学要点与正文一致性 → 鼻音表核查 → 版本同步
```

### 4.2 落地过渡环节设计标准

**规则**：任何高能量段落结束后，在引导睁眼前，必须包含生理落地过渡。

**标准过渡模板**：

```
[停顿 Xs——音乐渐柔]

先动动手指......
感受一下身体坐在这里的重量。

[停顿 Xs——钢琴温柔延续]
```

**设计原则**：
- 从内在体验→身体感知→外部环境，分三步过渡
- 不能直接从高能量宣告跳到"睁开眼睛"
- 动作指令从小到大：手指→手掌→肩膀→全身
- 停顿给足时间，让神经系统自然切换

### 4.3 体位中性化表达原则

**核心规则**：所有身体指令必须同时适用于椅子坐姿和盘腿坐姿。

**全文一致性检查**：

| 检查位置 | 要求 |
|----------|------|
| 理论基础段 | 锚点描述中性化 |
| 引导词正文 | 所有身体指令中性化 |
| 安全退出通道 | 回到身体的指令中性化 |
| 教学要点/常见反应 | 回应策略中性化 |
| 收束段 | 落地过渡中性化 |

**注意**：隐喻性表达（如"存入双脚——前行"）可保留，因为这是意象而非物理指令。需加注释说明。

### 4.4 引号使用规范

**核心规则**：冥想引导词中禁止使用嵌入式引号。

**原因**：
- 录音时引号内外的语调变化对听众而言难以区分
- 嵌入式引号在耳听体验中会造成认知干扰
- 增加录音师对"谁在说话"的判断负担

**转化规则**：

| 原始写法 | 正确写法 | 处理方式 |
|----------|----------|----------|
| 那个说"你不行"的声音 | 那个总说你不行的声音 | 去引号融入句子 |
| 告诉自己"我可以" | 在心里确认——我可以 | 用破折号+独立句 |
| 心里想着"如果失败怎么办" | 心里冒出那些还没来的事 | 间接引语重写 |
| 你会发现"原来我可以" | 你会发现——原来你可以 | 破折号+人称转换 |

### 4.5 语义准确性要求

**每个形容词/副词必须满足**：
1. **不可替换性**：换任何近义词都会丢失信息
2. **语境唯一性**：在此处只有这个词最精准
3. **口语节奏感**：适合当前语速和呼吸节奏
4. **鼻音友好**：优先选择发音清晰的字词

**实践案例**：

| 原词 | 优化词 | 优化理由 |
|------|--------|----------|
| 更真实 | 更笃定 | "真实"语义弱、与"安静""清醒"不构成递进；"笃定"含力量确认感 |
| 持续 | 不停 | "不停"更口语、节奏干脆、构成三重句式 |
| 本身就是力量的表现 | 本身就是力量 | 去"的表现"更直接、更有力 |
| 还没发生的担心 | 那些还没来的事 | "还没来"更口语、"事"比"担心"更中性 |

### 4.6 Unicode编码注意事项

**实践教训**：在编辑过程中需警惕形近字的Unicode混淆：

| 正确 | 错误 | Unicode差异 |
|------|------|-------------|
| 冥 (U+51A5) | 冒 (U+5192) | 上部件不同 |
| 挺 (U+633A) | 挚 (U+631A) | 右部件不同 |
| 熬 (áo) | 熆 | 形近误认 |
| 躲 (duǒ) | 蹲 | 形近误认 |
| 膛 (táng) | 膨 | 形近误认 |

**防范措施**：
- 关键字词完成后用 `xxd` 或 hex dump 核查字节编码
- 发现"无变化"报错时优先怀疑编码问题
- 鼻音表中的汉字逐一核对拼音是否匹配

# 邮件/微信/会议纪要版模板库 (Email, WeChat & Meeting Minutes Templates)

> **定位**：提供适用于邮件、即时通信和会议纪要的实用模板，方便在不同渠道中做向上沟通与风险留痕
> **核心原则**：短、准、可执行、可转发、可追踪

## 一、渠道使用原则

### 1.1 邮件适合
- 重要决定确认
- 跨部门协同
- 涉及责任边界和时间点
- 需要正式留痕的风险说明

### 1.2 微信/即时通信适合
- 快速同步进展
- 简短确认和追问
- 临时风险提醒
- 会前会后补充说明

### 1.3 会议纪要适合
- 统一多人理解
- 固化优先级、责任人、时间点
- 降低“我不是这个意思”的空间

## 二、邮件模板

### 2.1 日常进展同步邮件

```text
主题：X项目进展同步｜截至3月30日

王总，您好：
同步一下X项目当前进展：
1. 已完成：________；
2. 进行中：________；
3. 当前风险：________；
4. 下一步计划：________；
5. 需要您支持：________。

如您同意，我将按以上节奏推进，并于________再次同步。
谢谢。
```

### 2.2 风险预警邮件

```text
主题：X项目风险预警及建议方案

您好：
当前X项目出现一项需要提前处理的风险。

一、事实
________

二、影响
若24小时内未处理，可能影响________。

三、建议方案
A：________
B：________

四、我的建议
我建议优先选择________，原因是________。

如您同意，我将立即推进并同步相关方。
```

### 2.3 会后确认邮件

```text
主题：关于今日X会议结论的确认

您好：
根据今天会议，当前确认如下：
1. 当前目标：________；
2. 优先级排序：________；
3. 负责人：________；
4. 时间点：________；
5. 风险与依赖：________；
6. 待协调事项：________。

如无异议，我将按上述内容执行。
```

### 2.4 请求资源支持邮件

```text
主题：关于X项目资源支持的申请

您好：
为确保X项目按________完成，目前存在以下资源缺口：
- 缺口内容：________；
- 当前影响：________；
- 若不补足：________。

我建议两个方案：
1. ________
2. ________

请您确认优先方案，我将据此推进。
```

## 三、微信/即时通信模板

### 3.1 简短进展同步

```text
王总，汇报一下：
- 已完成：________
- 当前卡点：________
- 风险：________
- 我下一步：________
- 需要您确认：________
```

### 3.2 临时风险提醒

```text
先同步一个风险：________。
如果今天________前还未解决，可能影响________。
我已先做A动作，同时准备B方案。
如您同意，我会同步相关方并按B预案推进。
```

### 3.3 优先级确认

```text
收到。为避免理解偏差，我确认一下当前顺序：
1. 先做________
2. 暂缓________
3. 风险是________
我将按这个顺序推进，如有调整请直接告诉我。
```

### 3.4 催确认但不冒犯

```text
我补充确认一下，方便团队继续推进：
目前是按A方案执行，还是改为B方案？
如果您今天________前方便确认，我这边就能按时推动下去。
```

### 3.5 会后即时留痕

```text
刚才会议我先简要确认一下：
- 目标：________
- 负责人：________
- 时间点：________
- 已知风险：________
我稍后会补一版完整纪要。
```

### 4.1 标准版

```text
会议时间：________
参会人员：________

一、会议结论
1. ________

二、任务分工
- A负责人：________，完成时间：________
- B负责人：________，完成时间：________

三、风险与依赖
- 风险1：________
- 风险2：________

四、待确认事项
- ________

五、下次同步时间
- ________
```

### 4.2 高风险场景版

```text
会议主题：________
会议时间：________
参会人员：________

一、当前唯一优先级
________

二、已确认决定
1. ________
2. ________

三、责任边界
- 我方负责：________
- 对方负责：________

四、已知风险
- ________

五、若风险发生的应对方案
- ________
```

## 五、三种渠道如何配合使用

推荐节奏：
1. 先用微信快速同步和催确认
2. 关键决定改用邮件正式确认
3. 多人协同时补会议纪要统一口径

## 六、常见错误

- 微信里发太长，重点淹没
- 邮件只写背景，不写请求
- 纪要只记讨论，不记结论和责任
- 关键变更没有二次确认
- 口头已变更，但书面版本没更新

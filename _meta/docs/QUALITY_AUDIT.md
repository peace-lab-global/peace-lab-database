# Quality Audit Process | 季度质量审计流程

> 本文档定义 Peace Lab Database 的季度质量审计流程。
> 每季度末运行一次,生成审计报告,识别需要改进的领域。

---

## 一、审计目标 | Audit Goals

### 1.1 核心目标

1. **量化项目健康度**:6 个核心维度的可测量指标
2. **识别趋势变化**:与上季度对比,了解是改善还是退化
3. **指导治理决策**:基于数据决定优先级
4. **生成可执行清单**:列出需修复的具体问题

### 1.2 审计频率

- **季度审计**(完整):每季度一次,生成完整报告
- **月度审计**(简版):每月一次,检查关键指标
- **实时监控**(自动化):CI 检查新增内容

## 二、审计维度 | Audit Dimensions

### 2.1 结构维度(20%)

| 指标 | 计算方法 | 健康度阈值 |
|---|---|---|
| **INDEX 覆盖率** | 含 INDEX 的目录 / 含子目录的目录 | ≥ 95% |
| **INDEX 直属文件覆盖** | INDEX 列出文件数 / 实际文件数 | ≥ 95% |
| **跨引用健康度** | 1 - (broken_refs / total_refs) | ≥ 99% |
| **元文档完整性** | _meta/docs/ 文档数 | ≥ 8 |

### 2.2 元数据维度(15%)

| 指标 | 计算方法 | 健康度阈值 |
|---|---|---|
| **frontmatter 完整** | 完整文件数 / 总文件数 | ≥ 95% |
| **tags 覆盖率** | 有 tags 的文件 / 总文件 | ≥ 95% |
| **tags 层次合规** | 符合三层 tags 规则的文件 | ≥ 80% |
| **DOI 引用** | DOI 总数 | ≥ 100 |

### 2.3 内容深度维度(15%)

| 指标 | 计算方法 | 健康度阈值 |
|---|---|---|
| **中位深度** | 中位行数 | ≥ 100 |
| **深度内容比例** | > 500 行的文件 / 总文件 | ≥ 7% |
| **浅内容比例** | < 30 行非 INDEX 文件 / 总文件 | ≤ 5% |

### 2.4 学术严谨性维度(20%)

| 指标 | 计算方法 | 健康度阈值 |
|---|---|---|
| **有引用文件比例** | 有引用的文件 / 总文件 | ≥ 40% |
| **核心文档引用数** | 临床核心文档平均引用 | ≥ 5 |
| **DOI 覆盖率** | 有 DOI 的引用 / 总引用 | ≥ 30% |
| **参考文献段覆盖率** | 含 References 段的文件 / 核心文档 | ≥ 80% |

### 2.5 合规性维度(20%)

| 指标 | 计算方法 | 健康度阈值 |
|---|---|---|
| **免责声明覆盖** | 临床文件含免责声明 / 临床文件 | ≥ 95% |
| **危机资源覆盖** | 自杀提及文件含危机资源 / 自杀提及文件 | ≥ 95% |
| **危机热线引用** | 包含至少一个国家热线的文件 | ≥ 80% |

### 2.6 可发现性维度(10%)

| 指标 | 计算方法 | 健康度阈值 |
|---|---|---|
| **可发现文件比例** | 1 - (孤儿文件 / 总文件) | ≥ 99% |
| **mirror 标注率** | 已标 mirror 的副本 / 重复副本 | ≥ 95% |
| **stub 标记率** | 已标 stub 的骨架文件 / 骨架文件 | ≥ 80% |

## 三、审计流程 | Audit Workflow

### 3.1 季度审计步骤

```bash
# 1. 运行完整审计脚本
python3 Tools/scripts/quality_audit.py --output reports/quality-audit-Q4-2026.md

# 2. 生成 INDEX 完整性报告
python3 Tools/scripts/audit_index.py

# 3. 生成元数据健康度报告
python3 Tools/scripts/audit_metadata.py

# 4. 生成学术严谨性报告
python3 Tools/scripts/audit_citations.py

# 5. 生成合规性报告
python3 Tools/scripts/audit_compliance.py

# 6. 汇总并对比上季度数据
python3 Tools/scripts/audit_compare.py
```

## 五、紧急问题识别标准 | P0 Identification

### 5.1 触发 P0 的条件

- 免责声明覆盖率 < 80%(临床文件)
- 危机资源覆盖率 < 90%(自杀提及文件)
- broken_refs > 10(影响内容导航)
- DOI 总数下降(学术严谨性退化)
- 任何新增严重合规风险

### 5.2 触发 P1 的条件

- tags 过载(high-freq tags > 1000)且未拆分
- 镜像副本未标注 > 10 个
- 骨架文件比例 > 10%
- 浅内容比例上升 > 5%

### 5.3 触发 P2 的条件

- 元文档未更新
- 季度对比显示退化
- 任何新兴风险
